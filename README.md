# ZH-predictor

```
Para resolver o case GauchaZH, utilizou-se a metodologia CRISP DM, seguindo os seguintes passos: (A) Entendimento do problema; (B) Compreensão dos dados; (C) Preparação dos dados; (D) Modelagem; (E) Avaliação do Modelo; e (F) Implementação do modelo. 
```

## Project Structure

```
- environment.yml: arquivo de configuração do ambiente.

- requirements.txt: Libs necessárias para o projeto.

- data/: diretório de dados. Os dados brutos são salvos em data/raw, e os dados processados são salvos em data/processed 

- zhpred/zhpred: promove o modelo baseado em xgboost

```
